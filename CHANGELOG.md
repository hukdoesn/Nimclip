# Nimclip 更新日志

## [1.1.5]

- 修复约 500 条历史记录时滚动明显卡顿、界面可能无响应的问题，改用可复用的原生列表行，并减少列表状态的重复刷新。
- 图片缩略图和 Option 大图预览改为后台降采样解码，不再在主线程直接加载完整原图。
- 大图预览仅缓存当前一张降采样图片，限制缓存内存，并在关闭预览后立即释放；快速切换记录时会取消已经失效的解码任务。
- 修复 Option 预览可能因搜索框焦点或窗口状态变化而失效的问题。
- 优化图片 OCR：保持本地串行识别，同时避免每识别一张图片都重新读取全部历史记录，并及时释放识别过程中的临时图像内存。
- 优化新图片写入时的缩略图生成，避免先将高分辨率原图完整解码到内存。
- 图片原文件已经单独保存，不再把 PNG、TIFF 等图片表示作为富格式数据重复写入 SwiftData；升级后会自动清理旧图片记录中的冗余副本，进一步降低内存与数据库占用。
- 将组合键物理状态检测移出主线程，保留任意按键顺序与用户自定义快捷键支持，降低对界面响应的影响。
- 在设置与 README 中补充历史上限规则：超过上限会删除最旧的未收藏记录及其本地图片；收藏不计入上限且不会自动清理。

## [1.1.4]

- 修复默认及用户自定义全局快捷键依赖按键先后顺序的问题。
- 无论先按主键还是修饰键，只要当前配置的组合键同时处于按下状态即可唤起剪贴板历史。
- 同一轮长按只触发一次，松开任一所需按键后才能再次触发，避免重复弹出。
- 保留 macOS 原生全局热键注册路径，同时使用轻量按键状态检测兼容任意按键顺序。

## [1.1.3]

- 新增完全在本机运行的图片文字识别，可使用图片中的文字搜索历史图片。
- 支持自动识别新复制的图片，并可为已有历史图片批量补建一次文字索引。
- 修复点击状态栏图标时系统顶部菜单栏抖动的问题。
- 每 10 分钟自动检查一次新版本，并避免同一版本频繁提醒。
- 每次打开剪贴板面板时自动回到当前结果顶部和最新记录。
